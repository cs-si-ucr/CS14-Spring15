volatile - denotes to the compiler that this value may change - so that it doesn't attempt to optimize it.

The main usecase is when your program is interfacing with hardware and the hardware expects data to be laid out in a certain way.

const-correctness: Part 1

Declares x to be a "a const object of value type int"

const int x = 42;

or equivalently

int const x = 42;

Either of the above will cause any of the following lines to cause a compile time error.

x = 7;
int * p = &x;

However, the following line is ok:

const int *p = &x;

The line const int *p allows p to point to objects of type int that are constant, p can't be used to change the int object it's pointing to.

The line int * const p means that "p is a const pointer to an int that is non-const": you can't change the pointer p itself, but you can change the int object via p.
